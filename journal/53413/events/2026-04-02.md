## 2026-04-02 14:55

- Fixed a bug reported by Max: import.sql is not executed when using only StatelessSession with no Entity classes. Issue https://github.com/quarkusio/quarkus/issues/53413
- Root cause: SchemaManagementToolCoordinator iterates over metadata.getContributors() to decide which schema actions to run. With no entities, tables, or sequences, the contributor set is empty, so the coordinator skips everything including import.sql execution.
- Initial fix was 40 lines of manual schema creator invocation in SessionFactoryObserverForSchemaExport. After iteration, found a single-line fix: override getContributors() in PrevalidatedQuarkusMetadata to return Set.of("no-entities") when the set is empty, so the coordinator always processes configured schema actions.
- The contributor name does not matter because SchemaManagementToolCoordinator falls back to the root (non-contributor-qualified) action setting as the default.
